# China Employee Termination 2026: Pulse Brief on Evidence Decay for Foreign Employers

## Key takeaways
- In China terminations, evidence quality can decay quickly if records are not refreshed before key decisions.
- Foreign employers should manage **evidence decay** as a core risk, not a documentation detail.
- Route, payroll, and communication decisions become fragile when they rely on stale evidence.
- If critical evidence crosses freshness limits, pause and recertify before notice.
- This brief is Pulse-style and impact-first; legal depth is in the appendix.

## What changed and why this matters now
Teams often assume evidence collected earlier remains reliable through execution. In real workflows, facts evolve, records are updated, and assumptions shift. Without freshness checks, “approved” decisions can rest on outdated inputs.

This run rotates to an **evidence-decay format** (different from assumption-stability and prior models). It gives foreign operators a practical framework to protect decision quality over time.

## Plain-language legal context first
China employer-initiated termination generally requires lawful route selection and fact-consistent execution. For non-local operators, that means evidence must be current enough to support live decisions. Stale evidence can undermine legal position, payroll accuracy, and communication credibility.

Because local interpretation can vary, locality-sensitive evidence should also be refreshed when facts change.

## Pulse evidence-decay board

## Decay Zone 1 — Route evidence decay
**Risk:** route memo references facts that are no longer current.

**Freshness rule:** route basis must be reconfirmed before notice window.

**Decay trigger:** material fact changes without route memo refresh.

## Decay Zone 2 — Chronology evidence decay
**Risk:** contradiction logs become outdated as new records arrive.

**Freshness rule:** chronology baseline updated at each transition.

**Decay trigger:** unresolved contradictions older than SLA.

## Decay Zone 3 — Payroll/tax evidence decay
**Risk:** severance/IIT worksheet diverges from latest approved facts.

**Freshness rule:** final worksheet timestamp must align with latest legal/evidence baseline.

**Decay trigger:** multiple worksheet versions near communication.

## Decay Zone 4 — Communication evidence decay
**Risk:** scripts drift from latest approved assumptions.

**Freshness rule:** script recertification required after material updates.

**Decay trigger:** manager prep uses superseded messaging.

## Decay Zone 5 — Closure evidence decay
**Risk:** closure pack misses latest decisions and approvals.

**Freshness rule:** retrieval test on final version set only.

**Decay trigger:** archive index not updated after changes.

## Why this model helps foreign operators
Evidence-decay framing works well for distributed multinational teams because it turns “document quality” into timing-based controls. It is easier for leadership to govern freshness windows than to audit every file detail.

For executive oversight, a freshness dashboard provides clear signals: current, aging, stale, and blocked.

## Practical scenarios

### Scenario A: route evidence ages out before notice
Route approved earlier, but new facts arrive and memo is not refreshed.

**Outcome:** Decay Zone 1 red.

**Action:** revalidate route and pause downstream communication.

### Scenario B: payroll worksheet stale at release point
Payroll file finalized, then contradiction log updates later.

**Outcome:** Decay Zone 3 red.

**Action:** regenerate worksheet on latest baseline before notice.

### Scenario C: script freshness failure
Communication script approved two days earlier, then legal wording changes.

**Outcome:** Decay Zone 4 red.

**Action:** recertify script pack and re-brief spokesperson.

## Daily operations SOP (freshness governance)

## Step 1 — Define freshness windows by evidence type
- Set max age thresholds for route, chronology, payroll, and scripts.
- Assign owners and backups.

## Step 2 — Run freshness checks at transitions
- Mark evidence current/aging/stale.
- Log timestamps and dependencies.

## Step 3 — Enforce stale-evidence holds
- Stale critical evidence blocks communication.
- Trigger refresh workflow with SLA.

## Step 4 — Recertify after refresh
- Confirm alignment across legal/evidence/payroll/communication.
- Update version registry.

## Step 5 — Close with freshness audit
- Verify final records are current and retrievable.
- Capture recurring decay patterns.

## Required document checklist
- Route memo + fallback route + refresh timestamp
- Contract and amendment records
- Chronology ledger + contradiction log
- Local interpretation note
- Final severance/IIT/social worksheet + version timestamp
- Payment approvals + proof plan
- Script package + recertification record
- Closure memo + archive index + retrieval test
- Freshness log (domain, age, status, owner, action)

## Exception handling playbook
- **Route evidence stale near notice:** legal revalidation and hold.
- **Chronology evidence aging with open contradictions:** evidence sprint activation.
- **Payroll worksheet stale in final window:** automatic no-communication rule.
- **Script staleness detected:** communication recertification mandatory.
- **Closure staleness post-close:** reopen archive process and rerun retrieval.

## System implementation notes
Implement freshness timers in workflow tools with auto-alerts and stale-state holds. Link each critical decision to required evidence timestamp. Block communication when Zone 1-3 evidence is stale.

Track monthly KPIs: stale events by zone, refresh turnaround time, and dispute correlation.

## Common mistakes and prevention
- **Mistake:** assuming old approvals remain valid automatically.  
  **Prevention:** explicit freshness thresholds and recertification triggers.
- **Mistake:** version sprawl in payroll worksheets.  
  **Prevention:** one controlled final version with timestamp governance.
- **Mistake:** script updates outside control process.  
  **Prevention:** mandatory script freshness checks.
- **Mistake:** closure without freshness audit.  
  **Prevention:** final freshness gate before close.

## FAQ

### 1) What is evidence decay?
When previously valid records become outdated for current decisions.

### 2) Why is it risky?
Stale evidence undermines legal, payroll, and communication consistency.

### 3) Which decay zone is most sensitive pre-notice?
Payroll/tax evidence freshness (Zone 3).

### 4) Can we proceed with aging evidence?
Only with formal acceptance; stale critical evidence should block progress.

### 5) How often should freshness be checked?
At every major transition and material fact change.

### 6) Does this create more process overhead?
It reduces rework by preventing late-stage failures.

### 7) Is this legal advice?
No. It is an operational governance framework.
