# China Employee Termination: Pulse Traffic-Light Dispatch Brief for Foreign Employers (Hourly Testing Run)

- For foreign employers, China termination control improves when cases are managed through a strict traffic-light dispatch model: green to proceed, amber to fix, red to stop.
- Most escalation costs still come from handoff inconsistency between legal, HR, manager, and payroll teams.
- A traffic-light dispatch format gives leadership immediate clarity on what can move now and what must pause.
- Negotiated exits remain useful, but only if settlement drafting, payroll treatment, and communication records stay synchronized.
- City-level implementation variance remains a real uncertainty; local validation is still required before final release.

## Plain-language legal baseline (first)

China is not an at-will termination environment. Employer-initiated termination should align with a lawful route under the Labor Contract Law framework and be executed with coherent evidence and procedural consistency. For foreign teams, the operational sequence is straightforward:

1. Confirm lawful route on current facts.
2. Validate evidence chain quality for that route.
3. Ensure communication and settlement execution follow the same route logic.

State Council implementation regulations and SPC labor-dispute interpretation reinforce that disputes often hinge on consistency of records and process conduct, not just intent.

## Pulse dispatch board: traffic-light rules

### Green lane — proceed under standard controls

Criteria:
- Route note is current and consistent.
- Evidence chronology is coherent.
- Script and settlement records are aligned.
- Payroll treatment assumptions are validated.

**Action:** proceed with release plan and routine monitoring.

### Amber lane — pause for targeted correction

Typical amber triggers:
- minor route wording drift,
- unresolved chronology gap,
- script version ambiguity,
- settlement detail not fully mapped.

**Action:** assign owner + deadline; no employee-facing release until amber clears.

### Red lane — immediate stop/go hold

Red triggers:
- route cannot be defended on current facts,
- material evidence contradiction,
- settlement/payroll mismatch near lock,
- unresolved local implementation caveat.

**Action:** block communication and payout release; escalate to legal + payroll decision owners.

## 8 headline signals for foreign leadership

### 1) Route consistency signal
If route labels differ across teams, case confidence is overstated.

### 2) Evidence continuity signal
If chronology ownership is unclear, defensibility weakens quickly.

### 3) Communication integrity signal
If script drift appears, trust and legal consistency erode.

### 4) Settlement coherence signal
If settlement terms differ by channel, dispute risk rises.

### 5) Payroll lock readiness signal
If assumptions stay open near lock, operational risk spikes.

### 6) Local validation signal
If city-level checks remain open, release confidence should stay amber/red.

### 7) Escalation visibility signal
If updates show status but no owner/ETA, intervention quality drops.

### 8) Closure integrity signal
If payment proof/archive is incomplete, case is not truly closed.

## Practical implications for foreign employers

A traffic-light model helps non-China leadership make faster and safer calls. It standardizes handoff language and prevents ambiguous “almost ready” status that often causes late-cycle errors. This improves predictability in both compliance outcomes and internal workload planning.

For cross-border operations, the biggest benefit is decision discipline: only green moves forward; amber/red must be remediated with accountable ownership.

## Daily dispatch cadence (20 minutes)

### Minute 0-5: classify each active case
- Set lane status (green/amber/red).
- Verify route/evidence baseline.

### Minute 5-10: communication + settlement alignment
- Confirm script version and settlement source sheet.
- Hold non-green cases.

### Minute 10-15: payroll + local checks
- Validate pre-lock readiness.
- Confirm city-level validation status.

### Minute 15-20: escalation broadcast
- Report only amber/red cases.
- Include owner, ETA, and next review time.

## SOP by role

### HR
1. Maintain dispatch board and lane status.
2. Block release for amber/red cases.
3. Log employee communication records.

### Legal
1. Own route defensibility decision.
2. Review route wording consistency.
3. Co-sign stop/go for red triggers.

### Manager
1. Provide fact-based updates.
2. Use approved script only.
3. Escalate new material facts immediately.

### Payroll/Finance
1. Validate settlement-to-payroll mapping.
2. Enforce pre-lock review for amber/red cases.
3. Produce payment proof for closure.

### Compliance/Ops
1. Audit lane changes and reasons.
2. Track repeat amber/red patterns by city/team.
3. Verify closure checklist completeness.

## Required document checklist

- Canonical route note
- Evidence chronology log
- Approved communication script
- Settlement source sheet
- Payroll treatment mapping
- Local validation memo
- Payment proof + closure archive checklist

SLA baseline:
- route note within 1 business day,
- chronology pass within 2 business days,
- settlement/payroll alignment before lock,
- closure archive within 2 business days after payment.

## Exception playbook

### Route uncertainty rises (red)
- Stop communications.
- Re-test route with updated facts.
- Reissue route note before restart.

### Evidence contradiction found (red)
- Freeze progression.
- Resolve chronology ownership/date conflicts.
- Resume only after verified pass.

### Settlement challenged by employee (amber/red)
- Use documented clarification channel.
- Reconcile legal text with payroll mapping.
- Update approved response language.

### Local caveat discovered late (red)
- Apply local validation hold.
- Confirm city execution feasibility.
- Re-plan release sequence.

### Payroll mismatch near lock (red)
- Trigger stop/go gate.
- Correct mapping and revalidate.
- Reconfirm communication consistency.

## Common mistakes and prevention

### Mistake: amber treated as optional
**Prevention:** amber blocks release until remediation closes.

### Mistake: multiple “final” files
**Prevention:** one single-source artifact per document class.

### Mistake: leadership sees timeline only
**Prevention:** lane status + owner + ETA in every update.

### Mistake: local check deferred to end
**Prevention:** include local validation before final release.

### Mistake: closure declared too early
**Prevention:** require full closure checklist and proof pack.
