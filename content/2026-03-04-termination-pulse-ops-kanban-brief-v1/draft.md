# China Employee Termination in 2026: Pulse Ops-Kanban Brief for Foreign Employers

## Key takeaways
- Use an ops-kanban model to control termination workflow risk in China.
- Put route/evidence/payroll in blocker-sensitive lanes.
- Keep legal context first, then convert to practical board movement rules.

## Plain-language legal context first
China is not at-will. Employer-initiated termination generally requires a lawful statutory route plus coherent evidence and process consistency under the Labor Contract Law framework. Core anchors: Labor Contract Law, Decree No. 535 implementation rules, and SPC labor-dispute interpretation.

## Pulse ops-kanban board

### Backlog (must define)
- Canonical route memo
- Evidence chronology log template
- Settlement/payroll mapping template
- City-validation memo template

### In Progress (high attention)
- Script acknowledgement collection
- Local interpretation confirmation
- Pre-lock payroll/tax reconciliation

### Blocked (no-go until resolved)
- Route rationale conflict
- Evidence contradiction
- Settlement/payroll mismatch

### Ready for Action
- Route validated
- Evidence reconciled
- Payroll/legal co-sign complete

### Done (closure only)
- Payment proof archived
- Closure evidence packet complete
- Case closure checklist passed

## Board movement rules
- Any blocked item in route/evidence/payroll prevents progression.
- In-progress items nearing payroll lock auto-escalate to blocked if unresolved.
- Done requires closure packet completeness, not just payment execution.

## Why this helps foreign employers
Kanban gives a simple visual operating model across time zones: what is waiting, what is blocked, and what can proceed. It reduces confusion in cross-functional handoffs.

## What improved vs previous run
Compared with decision-gatecards, this run improves ongoing execution visibility by turning binary checks into a live workflow state model with movement rules.

## Minimum documentation controls
- Canonical route memo
