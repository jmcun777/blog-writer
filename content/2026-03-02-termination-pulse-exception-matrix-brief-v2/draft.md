# China Employee Termination: Exception Matrix Brief for Foreign Employers (Hourly Testing Run)

## Key takeaways
- Foreign employers should treat China termination risk as an exception-management problem, not a static policy question.
- Most escalation events come from known exception types that can be pre-mapped to clear actions.
- Pulse-style exception matrices help leadership act quickly without sacrificing legal and payroll control quality.
- Legal route choice is necessary but insufficient if exception handling is weak.
- Keep legal depth in appendix; lead with practical exception controls.

## Pulse lead
This run uses an **exception matrix** structure: each high-frequency exception is paired with trigger signal, business impact, and mandatory response. For foreign teams, this is easier to run than long narrative guidance because ownership and next actions are explicit.

## Plain-language legal context first
China’s Labor Contract Law sets route boundaries and severance principles. State Council implementation regulations and SPC interpretation shape how disputes evaluate process fairness, documentary reliability, and execution consistency. Practical takeaway: a route can start valid but lose defensibility if unresolved exceptions are allowed to pass through execution.

## Exception matrix (trigger → impact → mandatory response)

### Exception 1: Route condition invalidated mid-case
- **Trigger:** new fact conflicts with route assumptions
- **Impact:** legal foundation weakens quickly
- **Mandatory response:** freeze case progression, rerun legal fit, update route memo

### Exception 2: Critical evidence contradiction
- **Trigger:** chronology mismatch or missing core record
- **Impact:** high dispute vulnerability
- **Mandatory response:** hold employee-facing step, assign remediation owner + SLA

### Exception 3: Script drift by manager
- **Trigger:** unapproved language or conflicting statement
- **Impact:** narrative inconsistency risk
- **Mandatory response:** corrected written record + legal impact review

### Exception 4: Settlement component ambiguity
- **Trigger:** payout mapping unclear or changed late
- **Impact:** post-signature dispute risk
- **Mandatory response:** relock worksheet with legal/payroll co-sign

### Exception 5: Tax-treatment uncertainty
- **Trigger:** unsettled tax logic for payout structure
- **Impact:** trust and compliance risk
- **Mandatory response:** validate assumptions with advisor, refresh explanation note

### Exception 6: Escalation authority gap
- **Trigger:** no empowered decision owner at cutover
- **Impact:** rushed or stalled execution
- **Mandatory response:** activate backup authority before next step

### Exception 7: Premature case closure
- **Trigger:** signed but payment proof/archive incomplete
- **Impact:** reopened operational/legal exposure
- **Mandatory response:** reopen status until closure gate evidence complete

## Practical implications for foreign employers
- **HQ:** request exception heatmap, not just route summary.
- **Regional HR:** enforce exception gates as hard blockers.
- **Payroll/finance:** co-own settlement and tax exceptions pre-commitment.
- **Managers:** treat communication control as compliance requirement.

## Scenario snapshots
### Scenario A: Exception 2 caught before meeting
Evidence contradiction surfaced pre-meeting; team paused and remediated file before proceeding. Outcome: lower dispute risk.

### Scenario B: Exception 4 detected in payroll review
Settlement wording and payroll map diverged late; relock prevented post-signature confusion. Outcome: cleaner closure.

### Scenario C: Exception 6 near cutover
Primary approver unavailable; backup owner activated under matrix rule. Outcome: avoided uncontrolled execution.

## Daily operations SOP (matrix-driven)
1. Open case and apply exception matrix baseline.
2. Mark active exceptions with owner, SLA, and severity.
3. Resolve critical exceptions before employee-facing actions.
4. Run legal/payroll joint check on Exceptions 4/5.
5. Log exceptions and responses in real time.
6. Escalate unresolved critical items automatically.
7. Close only after Exception 7 criteria pass.

## Required document checklist
- Route memo with assumptions
- Evidence chronology + gap log
- Script pack + briefing proof
- Settlement worksheet + tax note
- Escalation authority matrix
- Exception register (1-7)
- Meeting/exception logs
- Payment proof + reconciliation
- Closure audit report

## Exception handling playbook
- unresolved critical exception at cutoff → auto-hold
- route/evidence change → rerun legal + evidence review
- settlement change after approval → relock and reapprove
- authority unavailable → activate backup chain

## System implementation notes
Track fields:
- exception ID/status/severity

Controls:
- hard stop on unresolved critical exceptions

## Common mistakes and prevention
- **Mistake:** exception register used as notes only.  
  **Prevention:** enforce blocker logic in workflow.
- **Mistake:** late payroll involvement.  
  **Prevention:** joint review on settlement/tax exceptions.
- **Mistake:** closing on signature.  
  **Prevention:** proof-based closure gate.

## FAQ
### Is the exception matrix legally required?
No, it is an operational governance method.

### Can small teams use it?
Yes, start with Exceptions 1-5 and expand.

### Why this format for foreign clients?
It converts legal complexity into action-ready decision rules.

### Is negotiated termination always safer?
Not automatically; unresolved exceptions can still create disputes.

### When should external PRC counsel be mandatory?
High-risk unilateral routes, protected-status concerns, restructuring, and multi-city complexity.

## Governance cadence for exception control
For foreign HQ teams, the matrix should run on a fixed cadence. Weekly: 30-minute exception stand-up on active cases with blocker aging, owner accountability, and next-action deadlines. Monthly: 60-minute trend review across entities focused on repeat exceptions, closure delays, and settlement clarity incidents. This cadence turns exception logs into continuous control improvement instead of static documentation.

## KPI pack for leadership
Use a compact KPI set for fast oversight:
- critical-exception count by case stage
- overdue exception aging over SLA
- script-drift incident frequency
- settlement relock rate
- closure lag (signature to verified close)
- post-close reopen rate within 30 days

If two or more KPIs deteriorate for two consecutive cycles, trigger a cross-functional recalibration session with HR, legal, and payroll owners.
