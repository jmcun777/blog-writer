# In China Employee Termination: Pulse Go/No-Go Gate Brief for Foreign Employers (2026 Testing Run)

## Key takeaways
- A practical way to reduce China termination risk is to run a strict **go/no-go gate** before employee-facing steps.
- Legal route clarity should be gate #1, but evidence quality, communication coherence, and settlement readiness must also pass.
- Foreign employers often approve “go” based on urgency instead of threshold-based controls.
- Payroll and IIT readiness should be no-go blockers when unresolved.
- Conditional approvals can handle city-level variance without stopping all actions.

## Pulse lead: what changed in execution quality
For foreign employers, China termination risk remains less about legal awareness and more about approval discipline. Teams often have policy frameworks and outside advice, yet still move too quickly from intention to action without explicit readiness thresholds. That gap—between strategy and gate control—drives many avoidable disputes.

This run uses a **go/no-go gate format** to make approval decisions testable. Instead of broad readiness claims, each critical domain must pass objective checks before execution proceeds.

## Plain-language legal/regulatory context first
China’s Labor Contract Law provides the core route framework for termination, while implementation regulations and judicial interpretation influence practical dispute review. In plain terms: a route may look valid on paper, but if records, communications, or settlement behavior conflict with that route, defensibility weakens.

A go/no-go model works for foreign operators because it forces route legality and operational reality to be checked together.

## Go/No-Go gateboard
### Gate 1: Legal route gate
**Go criteria:** route memo with legal anchor, risk grade, and legal signoff.

**No-go triggers:** conflicting route descriptions or missing legal basis.

### Gate 2: Evidence gate
**Go criteria:** dated, objective, centralized records with coherent chronology.

**No-go triggers:** evidence gaps, backfill risk, fragmented ownership.

### Gate 3: Communication gate
**Go criteria:** aligned script, notice package, meeting-note template, controlled language ownership.

**No-go triggers:** manager improvisation risk or script-document mismatch.

### Gate 4: Settlement gate
**Go criteria:** locked payout model (severance, wages, leave, IIT, timing) with approvals.

**No-go triggers:** unresolved calculations, unclear IIT handling, unapproved timing.

### Gate 5: Exception gate
**Go criteria:** refusal protocol, escalation path, and override governance pre-defined.

**No-go triggers:** no contingency process for high-friction events.

## Practical implications for foreign employers
- Approval quality improves when gates are measurable.
- Cross-functional alignment becomes easier when each team owns a gate.
- Leadership gets cleaner visibility into risk blockers and remediation timing.
- City-level differences can be handled through conditional gate thresholds.

For multinational operators, this structure also reduces internal friction between headquarters and local teams: instead of debating whether a case “feels ready,” both sides can review objective gate evidence, agree on blockers, and align on remediation actions with clear timelines and accountable owners.

## Daily operations SOP (role-by-role)
### Step A: Open case and pre-gate scan (HR + legal)
Classify route, define required gate evidence, assign case owner.

### Step B: Run gate checks 1-3 (HR + manager + legal)
Complete route, evidence, and communication checks before scheduling final employee-facing steps.

### Step C: Run gate 4 (Payroll + finance + HR)
Finalize settlement and IIT model, lock version, capture approvals.

### Step D: Run gate 5 and launch decision (HR + legal)
Confirm exception path and issue formal go/no-go decision with approver.

### Step E: Execute and log (HR owner)
If go, execute with timestamped event log and controlled documentation.

### Step F: 48-hour assurance review (Legal + HR + payroll)
Confirm all gate artifacts, payment proof, and closure archive.

## Required document checklist
- Route memo + legal signoff
- Evidence index + integrity result
- Communication package approval
- Settlement worksheet + version ID
- IIT treatment note + payment timeline confirmation
- Exception protocol + escalation map
- Approval chain and event log
- Archive manifest and closure signoff

## Exception handling playbook
### Gate fails before planned execution
Hold action, remediate failed gate, rerun approval.

### Employee refusal occurs despite go decision
Activate refusal protocol and escalate per gate 5 map.

### Settlement issue appears after go
Suspend outward communication, relock gate 4, reapprove.

### HQ requests override of no-go
Require formal risk acceptance with named approver; defer otherwise.

## System implementation notes
Add go/no-go controls in HRIS/payroll workflow:
- Gate status fields (1-5) with pass/fail reasons
- Blocker severity tags
- Settlement model version and IIT logic fields
- Override log and approver trace
- Event-log linkage and closure lock

## Common mistakes and prevention
### Mistake: “soft go” approvals
**Prevention:** binary gate status with documented criteria.

### Mistake: gate ownership is unclear
**Prevention:** assign accountable owner per gate.

### Mistake: no-go overrides without traceability
**Prevention:** mandatory override log and authorization level.

### Mistake: lessons not fed back into criteria
**Prevention:** monthly gate-failure review cycle.

## FAQ
### Is a go/no-go model legally required?
No. It is a governance method to improve decision quality.

### Can a case proceed if one gate is yellow?
Only with predefined conditional criteria and documented approval.

### Why is payroll a separate gate?
Because settlement and IIT errors often trigger avoidable claims.

### Is this too rigid for urgent cases?
Urgent cases can use expedited gates, but no gate should be invisible.

### When should external PRC counsel be mandatory?
High-risk unilateral routes, restructuring events, refusal scenarios, protected-sensitivity cases, and multi-city complexity.
